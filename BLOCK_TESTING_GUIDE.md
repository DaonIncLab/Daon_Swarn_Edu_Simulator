# 블록 코딩 테스트 가이드

## 수정 사항
✅ UP/DOWN 방향 이동 기능 추가됨 (DroneSimulator.ts)

## 좌표계
- **X축**: 왼쪽(-) / 오른쪽(+)
- **Y축**: 뒤(-) / 앞(+)
- **Z축**: 아래(-) / 위(+) (고도)

## 테스트 순서

### 1. 기본 테스트
```
1. 모든 드론 이륙 (고도 2m)
2. 대기 2초
3. 모든 드론 착륙
```

### 2. 대형 변경 테스트
```
1. 모든 드론 이륙 (고도 3m)
2. 대형 설정: Grid (행 2, 열 2, 간격 2m)
3. 대기 3초
4. 대형 설정: Circle (행 2, 열 2, 간격 2m)
5. 대기 3초
6. 대형 설정: V자 (행 2, 열 2, 간격 2m)
7. 대기 3초
8. 모든 드론 착륙
```

### 3. 방향 이동 테스트 (6방향 모두)
```
1. 모든 드론 이륙 (고도 3m)
2. 대기 2초
3. 대형 이동: 앞으로 3m
4. 대기 2초
5. 대형 이동: 오른쪽 2m
6. 대기 2초
7. 대형 이동: 위로 2m (고도 5m로 상승)
8. 대기 2초
9. 대형 이동: 아래로 1m (고도 4m로 하강)
10. 대기 2초
11. 모든 드론 착륙
```

## 디버깅 방법

### 브라우저 콘솔 확인
1. F12 (개발자 도구 열기)
2. Console 탭 선택
3. 다음 로그 확인:
   - `[DroneSimulator]` - 드론 시뮬레이터 동작
   - `[Interpreter]` - 블록 실행
   - `[ExecutionStore]` - 실행 상태

### 확인할 로그
- ✅ "Move formation: forward by 3m" - 대형 이동 명령
- ✅ "Set formation: grid" - 대형 설정 명령
- ✅ "Takeoff all to 2m" - 이륙 명령
- ❌ 에러 메시지가 있다면 확인

## 예상 동작

### 대형 설정이 제대로 작동한다면:
- Grid → 드론들이 격자 모양으로 배치
- Circle → 드론들이 원형으로 배치
- V자 → 드론들이 V자 모양으로 배치
- Line → 드론들이 일렬로 배치

### 방향 이동이 제대로 작동한다면:
- 앞으로 → Y축 양의 방향
- 뒤로 → Y축 음의 방향
- 왼쪽 → X축 음의 방향
- 오른쪽 → X축 양의 방향
- 위로 → Z축 양의 방향 (고도 증가) ← **새로 추가!**
- 아래로 → Z축 음의 방향 (고도 감소) ← **새로 추가!**

## 문제 해결

### "대형이 안 바뀌는 것 같아요"
1. 콘솔에서 `Set formation:` 로그 확인
2. 대형 설정 후 **최소 2-3초 대기** 필요 (드론이 이동하는 시간)
3. 간격(spacing) 값이 너무 작으면 변화가 안 보일 수 있음 → 2~3m 권장

### "움직임이 불규칙해요"
1. 명령 사이에 **대기 블록** 추가 (2~3초)
2. 거리 값이 너무 크면 화면 밖으로 나갈 수 있음
3. 콘솔에서 드론 위치 확인

### "위/아래 이동이 안 돼요"
- **이제 수정되었습니다!** 페이지 새로고침 후 다시 테스트해보세요.
